Part 3 - Introduction to variable and type

Display something on the screen is great, but it's not doing to take us anywhere on it's own. The display is static and can't be changed:

Begin
Write("a") ;
Write("a") ;
Write("a") ;
Write("a") ;
End

With it's equivalent in Javascript:

console.log("a");
console.log("a");
console.log("a");
console.log("a");

this will write the word "a" 4 times in a row no matter what. Now let's have a look at variables.

1 - What are variables and how to use them

As the name imply, it is a data item that may take on more than one value during the runtime of the program. You can give it a name and it is possible to display it multiple times during the program and not have the same result. Think of it as a container: you can put a data in it and at any time you can take it out and put another data in. Let's look at how to use it : first we need to state that we want to make a variable 

var namedvar;

the keyword "var" allows the computer to undersand that we want it to make a variable named "namedvar". There are 3 main types of data :

- numbers (1, 2, 3.5) who are just numbers;

- strings ("a", "hello", "32") who are words composed of characters. they are surrounded by quotes so that the computer may understand what it is: it prevent confusion between the number 36 and the word 36;

- boolean who are variable that have only two values: True or false;

In Javascript you not need to specify what kind of data you want to store because it will understand what kind you want depending on what you want to do with it (more on that later). Now that we have a variable, let's give it a value :

var namedvar;
namevar = 1;

our namedvar now contains the number 1. The corresponding algorithm is : 

Algorithm Assign

variables 
  namedvar : number 

Beginning 
  namedvar <- 1;
End

We state that we will need a variable namedvar and then we give it the value 1. To write it on the screen, since it's not a sentence, we don't need the quotes around it. Let's try to write it :

var namedvar;
namevar = 1;
console.log(namedvar);
console.log(namedvar);
console.log(namedvar);
console.log(namedvar);

We get the value 1 writen 4 times... But if we were to change the value of namedvar after the first write :

var namedvar;
namevar = 1;
console.log(namedvar);
namevar = 2;
console.log(namedvar);
console.log(namedvar);
console.log(namedvar);

We get the value 1 written once, followed by the value 2 written 3 times! With variables, you can keep a value for later and even modify this value.

2 - Operators

Now that we have a way to save a value, we can modify it with Operators. Operators are like + or - in mathematics: they take values and they give back the result. If we look a numbers, for example, we have access to most matematical operations (we have + to add 2 numbers, - to substract a number to another...). Let's say we want to know how much 10 + 12 is :

var namedvar;
namevar = 10 + 12;
console.log(namedvar);

we get 22! Of course, these Operators may be used on variables :

var namedvar1;
var namedvar2;
var varfinale;
namedvar1 = 10;
namedvar2 = 12;
varfinale = namedvar1 + namedvar2;
console.log(varfinale);

Let's translate it into an algorithm to understand it : 

Algorithm Addition

variables 
  namedvar1 : number 
  namedvar2 : number 
  varfinale : number 

Beginning 
  namedvar1 <- 10;
  namedvar2 <- 12;
  varfinale <- namedvar1 + namedvar2;
  Write(varfinale);
End

We put 2 numbers in two variables, two "containers" and then we take it out of each variables, add them, and put the result in another variables. This does not remove the value from namedvar1 and namedvar2 : we only take a copie of what is inside the variable to work. This mean that you can put a value in a variable and use it without losing it unless you put something else in it's place.
